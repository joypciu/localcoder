LocalCoder CLI starter som standard [TUI](/docs/tui), når den kører uden nogen argumenter.

```bash
localcoder
```

Men den accepterer også kommandoer som dokumenteret på denne side. Dette giver dig mulighed for at interagere med LocalCoder programmatisk.

```bash
localcoder run "Explain how closures work in JavaScript"
```

---

### tui

Start LocalCoder terminalbrugergrænsefladen.

```bash
localcoder [project]
```

#### Flag

| Flag                                     | Kort | Beskrivelse                                                                  |
| ---------------------------------------- | ---- | ---------------------------------------------------------------------------- |
| <nobr><code>{"--continue"}</code></nobr> | `-c` | Fortsæt sidste session                                                       |
| <nobr><code>{"--session"}</code></nobr>  | `-s` | Sessions-id for at fortsætte                                                 |
| <nobr><code>{"--fork"}</code></nobr>     |      | Forgren sessionen ved fortsættelse (brug med `--continue` eller `--session`) |
| <nobr><code>{"--prompt"}</code></nobr>   |      | Spørg om at bruge                                                            |
| <nobr><code>{"--model"}</code></nobr>    | `-m` | Model til brug i form af provider/model                                      |
| <nobr><code>{"--agent"}</code></nobr>    |      | Agent hos bruge                                                              |
| <nobr><code>{"--port"}</code></nobr>     |      | Port at lytte på                                                             |
| <nobr><code>{"--hostname"}</code></nobr> |      | Værtsnavn at lytte på                                                        |

## Kommandoer

LocalCoder CLI har også følgende kommandoer.

---

### agent

Administrer agenter i LocalCoder.

```bash
localcoder agent [command]
```

---

### attach

Tilslut en terminal til en allerede kørende LocalCoder backend-server startet via `serve` eller `web` kommandoer.

```bash
localcoder attach [url]
```

Dette gør det muligt at bruge TUI med en ekstern LocalCoder backend. For eksempel:

```bash
# Start the backend server for web/mobile access
localcoder web --port 4096 --hostname 0.0.0.0

# In another terminal, attach the TUI to the running backend
localcoder attach http://10.20.30.40:4096
```

#### Flag

| Flag                                     | Kort | Beskrivelse                                                                       |
| ---------------------------------------- | ---- | --------------------------------------------------------------------------------- |
| <nobr><code>{"--dir"}</code></nobr>      |      | Arbejdsmappe til at starte TUI i                                                  |
| <nobr><code>{"--continue"}</code></nobr> | `-c` | Fortsæt den seneste session                                                       |
| <nobr><code>{"--session"}</code></nobr>  | `-s` | Sessions-id for at fortsætte                                                      |
| <nobr><code>{"--fork"}</code></nobr>     |      | Forgren sessionen ved fortsættelse (brug med `--continue` eller `--session`)      |
| <nobr><code>{"--password"}</code></nobr> | `-p` | Adgangskode til basic auth (standard: `LOCALCODER_SERVER_PASSWORD`)                 |
| <nobr><code>{"--username"}</code></nobr> | `-u` | Brugernavn til basic auth (standard: `LOCALCODER_SERVER_USERNAME` eller `localcoder`) |

### auth

Kommando til at administrere legitimationsoplysninger og login for udbydere.

```bash
localcoder auth [command]
```

---

#### login

LocalCoder drives af udbyderlisten på [Models.dev](https://models.dev), så du kan bruge `localcoder auth login` til at konfigurere API-nøgler for enhver udbyder, du vil bruge. Dette er gemt i `~/.local/share/localcoder/auth.json`.

```bash
localcoder auth login
```

Når LocalCoder starter op, indlæser den udbyderne fra legitimationsfilen. Og hvis der er nogen nøgler defineret i dine miljøer eller en `.env` fil i dit projekt.

---

#### list

Viser alle de godkendte udbydere som gemt i legitimationsfilen.

```bash
localcoder auth list
```

Eller den korte version.

```bash
localcoder auth ls
```

---

#### logout

Logger dig ud af en udbyder ved at rydde den fra legitimationsfilen.

```bash
localcoder auth logout
```

### mcp

Administrator Model Context Protocol-servere.

```bash

```

---

#### add

Tilføj en MCP-server til din konfiguration.

```bash
localcoder mcp add
```

Denne kommando vil guide dig gennem tilføjelse af enten en lokal eller ekstern MCP-server.

---

#### list

Liste over alle konfigurerede MCP-servere og deres forbindelsesstatus.

```bash
localcoder mcp list
```

Eller brug den korte version.

```bash
localcoder mcp ls
```

---

#### auth

Godkendt med en OAuth-aktiveret MCP-server.

```bash
localcoder mcp auth [name]
```

Hvis du ikke angiver et servernavn, bliver du bedt om at vælge blandt tilgængelige OAuth-kompatible servere.

Du kan også angive OAuth-kompatible servere og deres godkendelsesstatus.

```bash
localcoder mcp auth list
```

Eller brug den korte version.

```bash
localcoder mcp auth ls
```

---

#### logout

Fjern OAuth-legitimationsoplysninger for en MCP-server.

```bash
localcoder mcp logout [name]
```

---

#### debug

Fejlfinding af OAuth-forbindelsesproblemer for en MCP-server.

```bash
localcoder mcp debug <name>
```

### models

Liste over alle tilgængelige modeller fra konfigurerede udbydere.

```bash
localcoder models [provider]
```

Denne kommando viser alle tilgængelige modeller på tværs af dine konfigurerede udbydere i formatet `provider/model`.

Dette er nyttigt til at finde ud af det nøjagtige modelnavn, der skal bruges i [your config](/docs/config/).

Du kan eventuelt videregive et udbyder-id for at filtrere modeller efter den udbyder.

```bash
localcoder models anthropic
```

#### Flag

| Flag                                    | Beskrivelse                                                             |
| --------------------------------------- | ----------------------------------------------------------------------- |
| <nobr><code>{"--refresh"}</code></nobr> | Opdater modelcachen fra models.dev                                      |
| <nobr><code>{"--verbose"}</code></nobr> | Brug mere detaljeret modeloutput (inkluderer metadata som omkostninger) |

Brug flaget `--refresh` til at opdatere den cachelagrede modelliste. Dette er nyttigt, når nye modeller er blevet tilføjet til en udbyder, og du vil se dem i LocalCoder.

```bash
localcoder models --refresh
```

### run

Kør localcoder i ikke-interaktiv tilstand ved at sende en prompt direkte.

```bash
localcoder run [message..]
```

Dette er nyttigt til scripting, automatisering, eller når du vil have et hurtigt svar uden at starte hele TUI. F.eks.

```bash "localcoder run"
localcoder run Explain the use of context in Go
```

Du kan også vedhæfte til en kørende `localcoder serve`-instans for at undgå MCP serverens kolde opstartstider ved hver kørsel:

```bash
# Start a headless server in one terminal
localcoder serve

# In another terminal, run commands that attach to it
localcoder run --attach http://localhost:4096 "Explain async/await in JavaScript"
```

#### Flag

| Flag                                     | Kort | Beskrivelse                                                                         |
| ---------------------------------------- | ---- | ----------------------------------------------------------------------------------- |
| <nobr><code>{"--command"}</code></nobr>  |      | Kommandoen til at køre, brug besked til args                                        |
| <nobr><code>{"--continue"}</code></nobr> | `-c` | Fortsæt sidste session                                                              |
| <nobr><code>{"--session"}</code></nobr>  | `-s` | Sessions-id for at fortsætte                                                        |
| <nobr><code>{"--fork"}</code></nobr>     |      | Forgren sessionen ved fortsættelse (brug med `--continue` eller `--session`)        |
| <nobr><code>{"--share"}</code></nobr>    |      | Del sessionen                                                                       |
| <nobr><code>{"--model"}</code></nobr>    | `-m` | Model til brug i form af provider/model                                             |
| <nobr><code>{"--agent"}</code></nobr>    |      | Agent til brug                                                                      |
| <nobr><code>{"--file"}</code></nobr>     | `-f` | Fil(er), der skal vedhæftes til meddelelsen                                         |
| <nobr><code>{"--format"}</code></nobr>   |      | Format: standard (formateret) eller json (rå JSON hændelser)                        |
| <nobr><code>{"--title"}</code></nobr>    |      | Titel for sessionen (bruger trunkeret prompt, hvis der ikke er angivet nogen værdi) |
| <nobr><code>{"--attach"}</code></nobr>   |      | Tilslut til en kørende localcoder-server (f.eks. http://localhost:4096)               |
| <nobr><code>{"--password"}</code></nobr> | `-p` | Adgangskode til basic auth (standard: `LOCALCODER_SERVER_PASSWORD`)                   |
| <nobr><code>{"--username"}</code></nobr> | `-u` | Brugernavn til basic auth (standard: `LOCALCODER_SERVER_USERNAME` eller `localcoder`)   |
| <nobr><code>{"--dir"}</code></nobr>      |      | Mappe at køre i, eller sti på fjernserveren ved tilkobling                          |
| <nobr><code>{"--variant"}</code></nobr>  |      | Modelvariant (udbyder-specifik ræsonneringsindsats)                                 |
| <nobr><code>{"--thinking"}</code></nobr> |      | Vis tænkeblokke                                                                     |
| <nobr><code>{"--port"}</code></nobr>     |      | Port til den lokale server (standard til vilkårlig port)                            |

### serve

Start en hovedløs LocalCoder-server til API-adgang. Tjek [server docs](/docs/server) for den fulde HTTP-grænseflade.

```bash
localcoder serve
```

Dette starter en HTTP-server, der giver API-adgang til localcoder-funktionalitet uden TUI-grænsefladen. Indstil `LOCALCODER_SERVER_PASSWORD` for at aktivere HTTP grundlæggende godkendelse (brugernavn er standard til `localcoder`).

#### Flag

| Flag                                     | Beskrivelse                                      |
| ---------------------------------------- | ------------------------------------------------ |
| <nobr><code>{"--port"}</code></nobr>     | Port at lytte på                                 |
| <nobr><code>{"--hostname"}</code></nobr> | Værtsnavn at lytte på                            |
| <nobr><code>{"--mdns"}</code></nobr>     | Aktiver mDNS-opdagelse                           |
| <nobr><code>{"--cors"}</code></nobr>     | Yderligere browseroprindelse til at tillade CORS |


#### Flag

### export

Eksporter sessionsdata som JSON.

```bash
localcoder export [sessionID]
```

Hvis du ikke angiver et sessions-id, bliver du bedt om at vælge mellem tilgængelige sessioner.

---

### import

Importer sessionsdata fra en JSON fil eller LocalCoder del URL.

```bash
localcoder import <file>
```

Du kan importere fra en lokal fil eller en LocalCoder share URL.

```bash
localcoder import session.json
localcoder import https://opncd.ai/s/abc123
```

### upgrade

Opdaterer LocalCoder til den nyeste version eller en specifik version.

```bash
localcoder upgrade [target]
```

For at opgradere til den nyeste version.

```bash
localcoder upgrade
```

For at opgradere til en bestemt version.

```bash
localcoder upgrade v0.1.48
```

#### upgrade

| Flag                                   | Kort | Beskrivelse                                                      |
| -------------------------------------- | ---- | ---------------------------------------------------------------- |
| <nobr><code>{"--method"}</code></nobr> | `-m` | Installationsmetoden, der blev brugt; curl, npm, pnpm, bun, brew |

| `LOCALCODER_CONFIG_CONTENT`             | string  | Indbygget json-konfigurationsindhold                                  |
| `LOCALCODER_DISABLE_AUTOUPDATE`         | boolean | Deaktiver automatisk opdateringskontrol                               |
| `LOCALCODER_DISABLE_PRUNE`              | boolean | Deaktiver beskæring af gamle data                                     |

| `LOCALCODER_DISABLE_DEFAULT_PLUGINS`    | boolean | Deaktiver standard plugins                                            |
