يمكن تشغيل LocalCoder كتطبيق ويب داخل متصفحك، ليمنحك تجربة البرمجة القوية بالذكاء الاصطناعي نفسها دون الحاجة إلى terminal.

![LocalCoder Web - جلسة جديدة](../../../assets/web/web-homepage-new-session.png)

## البدء

ابدأ تشغيل واجهة الويب عبر تنفيذ:

```bash
localcoder web
```

يؤدي ذلك إلى تشغيل خادم محلي على `127.0.0.1` بمنفذ عشوائي متاح، ويفتح LocalCoder تلقائيا في المتصفح الافتراضي لديك.

:::caution
إذا لم يتم تعيين `LOCALCODER_SERVER_PASSWORD` فسيكون الخادم دون حماية. هذا مناسب للاستخدام المحلي، لكنه يجب أن يكون مُعينا عند إتاحة الوصول عبر الشبكة.
:::

:::tip[مستخدمو Windows]
لأفضل تجربة، شغّل `localcoder web` من [WSL](/docs/windows-wsl) بدلا من PowerShell. يضمن ذلك وصولا صحيحا إلى نظام الملفات وتكاملا أفضل مع terminal.
:::

## الإعدادات

يمكنك ضبط خادم الويب باستخدام خيارات CLI أو عبر [ملف الإعدادات](/docs/config).

### المنفذ

افتراضيا، يختار LocalCoder منفذا متاحا. يمكنك تحديد منفذ:

```bash
localcoder web --port 4096
```

### اسم المضيف

افتراضيا، يرتبط الخادم بـ `127.0.0.1` (للاستخدام المحلي فقط). لجعل LocalCoder متاحا على شبكتك:

```bash
localcoder web --hostname 0.0.0.0
```

عند استخدام `0.0.0.0` سيعرض LocalCoder كلا من عناوين الوصول المحلي وعناوين الشبكة:

```
  Local access:       http://localhost:4096
  Network access:     http://192.168.1.100:4096
```

### اكتشاف mDNS

فعّل mDNS لجعل خادمك قابلا للاكتشاف على الشبكة المحلية:

```bash
localcoder web --mdns
```

يضبط ذلك تلقائيا اسم المضيف إلى `0.0.0.0` ويعلن عن الخادم باسم `localcoder.local`.

يمكنك تخصيص اسم نطاق mDNS لتشغيل عدة نسخ على الشبكة نفسها:

```bash
localcoder web --mdns --mdns-domain myproject.local
```

### CORS

للسماح بنطاقات إضافية عبر CORS (مفيد للواجهات الأمامية المخصصة):

```bash
localcoder web --cors https://example.com
```

### المصادقة

لحماية الوصول، عيّن كلمة مرور عبر متغير البيئة `LOCALCODER_SERVER_PASSWORD`:

```bash
LOCALCODER_SERVER_PASSWORD=secret localcoder web
```

اسم المستخدم الافتراضي هو `localcoder`، ويمكن تغييره عبر `LOCALCODER_SERVER_USERNAME`.

## استخدام واجهة الويب

بعد التشغيل، تتيح لك واجهة الويب الوصول إلى جلسات LocalCoder الخاصة بك.

### الجلسات

اعرض جلساتك وأدرها من الصفحة الرئيسية. يمكنك رؤية الجلسات النشطة وبدء جلسات جديدة.

![LocalCoder Web - جلسة نشطة](../../../assets/web/web-homepage-active-session.png)

### حالة الخادم

انقر على "See Servers" لعرض الخوادم المتصلة وحالتها.

![LocalCoder Web - See Servers (عرض الخوادم)](../../../assets/web/web-homepage-see-servers.png)

## إرفاق terminal

يمكنك إرفاق واجهة terminal (TUI) بخادم ويب قيد التشغيل:

```bash
# ابدأ خادم الويب
localcoder web --port 4096

# في محطة طرفية أخرى، اربط TUI
localcoder attach http://localhost:4096
```

يتيح لك ذلك استخدام واجهة الويب وterminal في الوقت نفسه، مع مشاركة الجلسات والحالة نفسها.

---

## ملف الإعدادات

يمكنك أيضا ضبط إعدادات الخادم داخل ملف الإعدادات `localcoder.json`:

```json
{
  "server": {
    "port": 4096,
    "hostname": "0.0.0.0",
    "mdns": true,
    "cors": ["https://example.com"]
  }
}
```

تكون خيارات CLI ذات أولوية أعلى من إعدادات ملف الإعدادات.
